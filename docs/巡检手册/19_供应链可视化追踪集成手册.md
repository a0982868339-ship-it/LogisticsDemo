## 1. 概述

EMIS供应链可视化追踪（SC-Vis）模块将EMIS巡检数据与园区WMS、TMS（运输管理系统）、ERP及外部承运商数据整合，为各方参与者提供**跨境订单全生命周期**的实时追踪服务，并支持货物在园区内的实体位置与物流状态的精确关联。

---

## 2. 追踪数据维度

### 2.1 货物追踪数据层次

| 追踪层次 | 数据粒度 | 更新频率 | 数据来源 |
|---|---|---|---|
| **订单层**（Order Level）| 一个订单包含若干货物 | 每小时 | CBES平台、TMS |
| **运单层**（Waybill Level）| 一个运单对应一批货物 | 15分钟 | 承运商API、海关申报 |
| **包裹层**（Package Level）| 单个包裹（箱/件）| 5分钟 | RFID/条形码扫描、EMIS识别 |
| **货位层**（Location Level）| 包裹当前的实体货位坐标 | 实时（位置变更时）| EMIS机器人 + WMS |

### 2.2 追踪事件类型

| 事件代码 | 事件名称 | 触发方式 | 关键数据字段 |
|---|---|---|---|
| EVT-001 | 货物到达园区 | 闸口扫描 + M3机器人拍摄 | 车牌号、运单号、到达时间 |
| EVT-002 | 货物入库 | WMS入库操作 + RFID确认 | 货位、货物编号、入库时间、操作员 |
| EVT-003 | 巡检合规通过 | EMIS自动识别 | 机器人ID、置信度、通过时间 |
| EVT-004 | 巡检告警触发 | EMIS自动识别 | 告警类型、等级、证据链接 |
| EVT-005 | 海关查验开始 | 合规审计员操作 | 查验员工号、开始时间、查验类型 |
| EVT-006 | 海关查验完成 | 合规审计员操作 | 查验结论、完成时间 |
| EVT-007 | 货物移位 | WMS移位操作 + EMIS确认 | 原货位、新货位、移位原因 |
| EVT-008 | 货临时查扣 | 合规审计员操作 | 查扣编号、原因、经手人 |
| EVT-009 | 货物出库 | WMS出库 + 闸口扫描 | 出库时间、目的地、承运商 |
| EVT-010 | 货物损毁发现 | EMIS识别（破损检测）| 损毁程度评分、图像证据 |

### 3.3 区块链存证集成

所有EVT-003（合规通过）、EVT-004（告警）、EVT-005~006（海关查验）事件均写入**海南自贸港跨境贸易联盟链（HNTradeChain）**，确保证据不可篡改：

| 参数 | 配置 |
|---|---|
| 链类型 | 许可联盟链（Hyperledger Fabric 2.5）|
| 共识机制 | PBFT（拜占庭容错）|
| 通道名称 | `syzbq-inspection-channel` |
| 链代码名称 | `emis-inspection-cc` |
| 写入语言 | Go（ChaincodeStub接口）|
| 哈希算法 | SHA-256 |
| 参与节点 | 三亚海关（Peer1）、园区节点（Peer2）、海南省商务厅（Peer3）|
| 数据开放查询 | 企业通过链API（需身份准入）可查询自己货物的存证哈希 |

**存证示例：**

## 4. 数字孪生园区集成

### 4.1 概述

SC-Vis模块与园区**数字孪生平台（Digital Twin Platform，DTP）**深度集成，在三维虚拟园区中实时展示：

- 所有机器人的实时位置和移动轨迹
- 货物的实体货位（货架三维坐标）
- 告警事件的空间位置标注
- 人员流量热图（匿名化）
- 设备运行状态（温度色彩映射）

**数字孪生平台技术栈：**

| 组件 | 技术 | 说明 |
|---|---|---|
| 三维渲染引擎 | Unity 3D（Web版：WebGL）| 在浏览器中渲染3D园区模型 |
| 实时数据推送 | WebSocket（`wss://dt.syzbq.internal:8443`）| 位置坐标持续推送 |
| 园区BIM模型 | IFC格式（Revit导出）| 全园区1:1比例模型 |
| 空间位置分辨率 | ±5cm（对接机器人定位系统）| SLAM+UWB融合定位 |

## 5. SCVis系统关键性能指标

| 指标 | 目标值 | 描述 |
|---|---|---|
| 货物位置更新延迟 | ≤60秒 | 从实体位置变化到系统更新的端到端时延 |
| 追踪事件写入延迟 | ≤30秒 | 事件发生到API可查询的时延 |
| 区块链存证延迟 | ≤120秒 | 事件发生到链上确认的时延 |
| API可用性 | ≥99.9% | 企业API接口年度可用性 |
| 数字孪生刷新率 | 10Hz（机器人位置）/ 5Hz（货物状态）| 三维场景数据刷新率 |
| 并发查询支持 | ≥5,000并发（全园区企业峰值估算）| API网关限流策略 |

## 6. 错误码

| 错误码 | 描述 | 严重等级 | 处置措施 |
|---|---|---|---|
| SCV-E001 | 运单查询不到任何追踪记录 | 低 | 检查货物是否已入库，WMS同步是否正常 |
| SCV-E002 | 区块链节点连接失败 | 严重 | 本地缓存队列暂存，待节点恢复后批量提交 |
| SCV-E003 | 承运商API拉取失败 | 中等 | 切换备用拉取频率（降至每小时），记录告警 |
| SCV-E004 | 数字孪生WebSocket连接断开 | 中等 | 客户端自动重连（指数退避，3~30秒间隔）|
| SCV-E005 | 货位坐标与WMS记录不一致（差>1米）| 中等 | 触发货物位置复核，机器人到场确认 |
| SCV-E006 | 企业API Key超额限流 | 低 | 返回HTTP 429，引导企业优化调用频率 |
| SCV-E007 | 存证哈希验证失败（数据被篡改）| 严重 | 立即通报CISO，触发S2安全事件响应 |
| SCV-E008 | 数字孪生3D模型与实际地图版本不一致 | 中等 | 刷新DTP平台模型，同步最新HD-MAP |
