## 1. 概述

本规范定义EMIS系统与跨境电商监管平台的完整集成方案，包括国家跨境电子商务监管平台（以下简称"国监平台"）、海南自贸港跨境数据中心（HNFTZ-DC）、海南一网通跨境电商服务平台（eHainan），以及园区内部各业务平台的数据交换协议、业务流程及运维要求。

## 3. 对接一：国家跨境电商监管平台（CBES）

### 3.1 对接内容

| 数据流方向 | 数据类型 | 频率 | 用途 |
|---|---|---|---|
| EMIS → CBES | 巡检异常告警上报 | 实时（事件触发）| 海关异常通报 |
| EMIS → CBES | 货物识别核验结果 | 批量（每小时）| 三单比对辅助 |
| CBES → EMIS | 订单申报数据 | 实时拉取（按需）| 货物合规判定依据 |
| CBES → EMIS | 风险名单与禁限清单更新 | 每日推送（凌晨02:00）| AI模型动态更新 |

### 3.2 技术参数

| 参数 | 值 |
|---|---|
| 接口基础URL | `https://api.cbes.customs.gov.cn/open/v2` |
| 认证方式 | OAuth 2.0（客户端凭证模式）+ 数字签名（SM3WithSM2）|
| 令牌有效期 | 7200秒（2小时），支持刷新 |
| 数据格式 | JSON + GZIP压缩（Body > 10KB时强制压缩）|
| 字符编码 | UTF-8 |
| 时区 | Asia/Shanghai（UTC+8）|
| 签名算法 | 国密SM3WithSM2（优先）/ RSA-SHA256（备用）|
| 幂等机制 | 请求Header携带`Idempotency-Key`（UUID v4，72小时去重）|

## 4. 对接二：海南自贸港跨境数据中心（HNFTZ-DC）

### 4.1 对接内容与目标

HNFTZ-DC是海南省建立的跨境贸易数据汇聚平台，EMIS系统作为数据提供方，须按规定向其提供：

| 数据集名称 | 字段数量 | 提交频率 | 数据量（日）|
|---|---|---|---|
| 园区日巡检统计摘要 | 28个字段 | 每日 23:55 | 1条 |
| 货物识别汇总数据 | 42个字段 | 每日 23:55 | 约1.8万条 |
| 异常告警事件数据 | 36个字段 | 实时推送 | 约30~200条 |
| 机器人运营状态数据 | 15个字段 | 每小时 | 372条 |

### 4.2 数据格式（HNFTZ-DC）

采用**Apache Kafka**消息队列推送，EMIS作为Producer。

```
Kafka连接参数：
  Bootstrap Servers: kafka1.hnftz-dc.gov.cn:9093, kafka2.hnftz-dc.gov.cn:9093
  Security Protocol: SASL_SSL
  SASL Mechanism: GSSAPI (Kerberos)
  Keytab: /etc/security/keytabs/emis-syzbq.keytab
  Principal: emis/syzbq@HNFTZ.GOV.CN
  SSL Truststore: /etc/ssl/hnftz-ca.jks

Topic配置：
  hnftz.emis.inspection.stats      # 巡检统计
  hnftz.emis.goods.recognition     # 货物识别
  hnftz.emis.alert.events          # 异常告警
  hnftz.emis.robot.status          # 机器人状态
  
Topic参数（生产侧）：
  ack=all, retries=5, compression=lz4
  batch.size=65536 (64KB)
  linger.ms=50
```

## 5. 对接三：园区内部WMS系统

### 5.1 数据交换概述

WMS（仓储管理系统，SYZBQ-WMS v6.x）是EMIS最重要的内部对接系统。

| 接口 | 方向 | 协议 | 频率 |
|---|---|---|---|
| 库存货物清单同步 | WMS → EMIS | REST API（拉取）| 实时（5分钟一次增量）|
| 货位变更通知 | WMS → EMIS | Webhook推送 | 实时（事件驱动）|
| 货物识别结果写回 | EMIS → WMS | REST API | 实时（事件驱动）|
| 异常货物状态更新 | EMIS → WMS | REST API | 实时（事件驱动）|

## 6. 接口监控与SLA要求

| 接口 | 可用性SLA | 响应时间P99 | 错误率告警阈值 |
|---|---|---|---|
| CBES申报查询 | 99.5%（受外部限制）| ≤3s | >1% 触发告警 |
| CBES告警推送 | 99.99%（EMIS侧）| ≤2s | >0.1% 触发告警 |
| HNFTZ-DC Kafka生产 | 99.9% | P99<1s | 积压>1000条告警 |
| WMS货位同步 | 99.9%（内网）| ≤1s | >0.5% |
| WMS Webhook接收 | 99.99% | ≤500ms | >0.1% |

## 7. 接口错误码

| 错误码 | 来源系统 | 描述 | 处置措施 |
|---|---|---|---|
| PLT-E001 | CBES | OAuth令牌过期 | 自动刷新令牌 |
| PLT-E002 | CBES | 禁限清单拉取失败 | 保留上一版本，次日重试；通知合规部 |
| PLT-E003 | HNFTZ-DC | Kafka认证失败（Kerberos票据过期）| 自动重新获取票据（kinit）|
| PLT-E004 | HNFTZ-DC | Topic写入超时 | 本地队列暂存，等待网络恢复 |
| PLT-E005 | WMS | 货位同步延迟>10分钟 | 降级处理：关闭货位实时验证，切换为纯视觉识别 |
| PLT-E006 | WMS | Webhook接收失败 | 返回204重试信号，WMS最多重试3次，超限记录 |
| PLT-E007 | 通用 | 接口签名不匹配 | 检查密钥状态，可能需轮换密钥 |
| PLT-E008 | CBES | 申报单不存在 | 触发P1告警：无申报单货物 |
| PLT-E009 | 内部 | 数据脱敏服务不可用 | 暂停向HNFTZ-DC推送含个人信息字段，通知DPO |
