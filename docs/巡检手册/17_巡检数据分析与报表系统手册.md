## 1. 系统概述

EMIS-Analytics是EMIS系统的数据分析与报表子系统，整合来自调度引擎、AI识别引擎、传感器网络和外部接口的全量数据，为运营管理人员、合规审计人员和海关驻场人员提供实时看板、定期报表和数据导出服务。

**系统访问地址**：`https://emis-analytics.syzbq.internal:8443`  
**移动端App**：EMIS Manager（iOS/Android，仅限园区内网Wi-Fi访问）  
**数据刷新频率**：实时看板1分钟刷新；报表T+1日生成

## 2. 数据架构

### 2.1 数据流

```
数据源层                         处理层                           展示层
─────────────────────────────────────────────────────────────────────
调度引擎日志                  →   │                    │  →  实时看板
AI识别结果                    →   │  Apache Flink       │  →  日/周/月报
传感器数据（IoT）              →   │  流处理引擎         │  →  临时查询
机器人状态日志                →   │                    │  →  数据导出
海关接口响应日志               →   ├────────────────────┤
WMS同步数据                   →   │  ClickHouse OLAP   │
CBES申报数据                  →   │  列式存储引擎       │
                                  │  (3节点集群)        │
                                  └────────────────────┘
```

### 2.2 数据仓库架构

| 层级 | 技术 | 数据量（日增）| 保存策略 |
|---|---|---|---|
| 原始层（ODS）| Kafka + HDFS | 约2.4TB | 保留30天，压缩后归档 |
| 明细层（DWD）| ClickHouse | 约400GB | 热存3年，冷存10年 |
| 汇总层（DWS）| ClickHouse | 约20GB | 永久保留 |
| 应用层（ADS）| PostgreSQL | 约5GB | 永久保留 |
| 实时流（ODS-RT）| Apache Kafka + Redis | — | 24小时滑动窗口 |

## 4. 定期报表体系

### 4.1 日报（每日06:00自动生成）

**报表编号：** RPT-DAILY-001  
**收件人：** 智能运营中心主任、各仓储区域主管、海关驻场组组长（仅合规指标部分）  
**格式：** Excel + PDF（自动发送至邮件+OA系统）

| 报表章节 | 关键指标 |
|---|---|
| 机器人舰队运营 | 在线率（均值/最低）、任务完成率、故障台次 |
| 货物巡检统计 | 识别总量、通过数、复核数、扣留数、告警数 |
| 抽检覆盖情况 | 各区域抽检率、与目标差距 |
| 告警事件汇总 | P0/P1/P2/P3各等级数量及处置状态 |
| 冷链监控摘要 | 超温事件数、最大超温幅度和持续时间 |
| 系统健康指标 | DB响应时间、API成功率、存储使用率 |

### 4.2 周报（每周一08:00生成，覆盖上周Mon~Sun）

**报表编号：** RPT-WEEKLY-001  
**收件人：** 副总经理、部门全体主管  

额外于日报的内容：
- 周度趋势分析（同比/环比）
- Top10告警类型及频率分析
- 机器人预测性维护提醒（基于使用时间和故障频率推算）
- 人员违规行为汇总（匿名化）

### 4.3 月报（每月1日10:00生成，覆盖上月）

**报表编号：** RPT-MONTHLY-001  
**收件人：** 管理委员会全体、法务合规部、海关驻场组  
**特殊要求：** 月报须符合三亚海关2024年第3号公告要求，数据字段和格式须经合规审计部审核

月报包含11个固定章节：

| 章节 | 内容摘要 |
|---|---|
| 1. 执行摘要 | 月度运营总体情况（300字以内）|
| 2. 机器人舰队月度统计 | 在线率趋势、故障率、维修工单统计 |
| 3. 货物识别与合规统计 | 识别量、通过率、各品类分布、HS编码分析 |
| 4. 异常事件详情 | 所有P0/P1事件详细记录（含处置结论） |
| 5. 海关协作情况 | 告警通报数、海关处置结论、案件移交数量 |
| 6. 冷链专项报告 | 超温事件分析、货物损失评估 |
| 7. 数据安全合规 | 安全事件、访问审计摘要 |
| 8. KPI达成情况 | 8项KPI的月度达成率对比 |
| 9. 系统可用性报告 | 各接口SLA达成情况 |
| 10. 问题与改进 | 当月遇到的问题和采取的改进措施 |
| 11. 下月计划 | 维护计划、系统升级计划、特殊任务 |

## 5. 临时查询功能

### 5.1 查询语法

EMIS-Analytics支持类SQL的查询语言（EMIS-QL），示例：

```sql
-- 查询本周内高风险货物识别记录
SELECT
  robot_id,
  waybill_no,
  detected_hs_code,
  declared_hs_code,
  confidence,
  alert_level,
  created_time
FROM emis.goods_recognition
WHERE alert_level IN ('P0', 'P1')
  AND created_time >= '2025-10-27 00:00:00'
  AND created_time < '2025-11-03 00:00:00'
ORDER BY created_time DESC
LIMIT 1000;

-- 查询各区域日均温度趋势（近30天冷链区C）
SELECT
  DATE(sample_time) as date,
  zone_code,
  AVG(temperature_celsius) as avg_temp,
  MIN(temperature_celsius) as min_temp,
  MAX(temperature_celsius) as max_temp,
  COUNT(CASE WHEN temperature_celsius > 8.5 THEN 1 END) as over_threshold_count
FROM emis.temperature_log
WHERE zone_code LIKE 'CK-C%'
  AND sample_time >= NOW() - INTERVAL 30 DAY
GROUP BY DATE(sample_time), zone_code
ORDER BY date, zone_code;
```

### 5.2 查询权限矩阵

| 用户角色 | 可查询数据范围 | 最大导出行数 | 导出格式 |
|---|---|---|---|
| 数据分析师 | L1/L2（脱敏）| 100万行 | CSV/Excel |
| 合规审计员 | L1/L2/L3（指定字段）| 50万行 | CSV/Excel/PDF |
| AI算法工程师 | L1/L2（匿名化）| 500万行 | CSV/Parquet |
| 调度员 | L1/L2（本班次相关）| 1万行 | CSV |
| 海关驻场人员 | L2（指定字段）| 10万行 | PDF |
| 系统管理员 | 全量（含L3）| 不限 | 所有格式 |

## 6. 数据质量管理

### 6.1 数据质量检测规则

| 检测项目 | 规则 | 告警阈值 |
|---|---|---|
| 数据完整性 | 核心字段不能为NULL | 缺失率>0.1% 告警 |
| 数据及时性 | 任务记录从完成到入库时延 | >2分钟 告警 |
| 数据一致性 | 机器人上报的任务ID与调度引擎记录一致 | 不一致率>0.5% 告警 |
| 数据准确性 | 货物抽检结论与复核结论一致率 | <98% 告警 |
| 重复数据率 | 同一告警ID仅出现一次 | 重复率>0.01% 告警 |

### 6.2 数据质量错误码

| 错误码 | 描述 | 处置 |
|---|---|---|
| DQ-E001 | 批次数据管道延迟超过30分钟 | 检查Flink任务状态，联系IT |
| DQ-E002 | 核心字段缺失率超阈值 | 查找上游数据源问题，追溯修复 |
| DQ-E003 | 关键报表生成失败 | 手动触发重试，联系数据分析组 |
| DQ-E004 | ClickHouse集群磁盘使用率>80% | 扩容或归档历史数据 |
| DQ-E005 | 报表发送邮件失败 | 检查SMTP服务；手动推送OA系统 |
| DQ-E006 | 数据导出任务超时（>10分钟）| 缩小查询范围，分批导出 |
