## 2. 调度系统架构

### 2.1 核心组件

```
EMIS调度引擎 (EMIS-DISPATCH)
    ├── 全局路径规划器 (Global Path Planner, GPP)
    │       ├── 地图服务 (HD Map Service)
    │       ├── 任务分配算法 (MILP优化器)
    │       └── 动态禁区管理器 (Dynamic No-Go Zone Manager)
    ├── 局部避障控制器 (Local Obstacle Avoidance, LOA)
    │       ├── TEB (Timed Elastic Band) 规划器
    │       └── DWA (Dynamic Window Approach) 备用
    ├── 机器人舰队管理器 (Fleet Manager, FM)
    │       ├── 状态聚合器
    │       └── 冲突解决器 (Deadlock Resolver)
    └── 任务队列管理器 (Task Queue Manager, TQM)
            ├── 优先级排序引擎
            └── 历史任务存档器
```

## 3. 任务优先级模型

### 3.1 优先级分级

EMIS-DISPATCH采用**6级优先级**体系（P0最高，P5最低）：

| 优先级 | 名称 | 触发条件 | 响应时效 | 示例 |
|---|---|---|---|---|
| **P0** | 紧急安全 | 火警/气体泄漏/人员摔倒告警 | 立即中断所有任务，≤90s到场 | 火灾报警联动 |
| **P1** | 高优违规 | 货物标签异常、未申报货物 | ≤3min插队分配 | 海关疑点货物复核 |
| **P2** | 定时巡检（必检）| 海关协议巡检点、危化品区 | 按计划时间±5min | 危化品区每2h一巡 |
| **P3** | 常规巡检 | 标准仓储区例行巡检 | 按计划时间±15min | 普通仓库每4h一巡 |
| **P4** | 按需巡检 | 人工/系统触发的临时任务 | ≤15min分配 | 客户投诉后复查 |
| **P5** | 低优后台 | 地图更新采集、设备健康拍摄 | 有空闲机器人时执行 | 夜间增量建图 |

### 3.2 优先级抢占规则

- P0任务触发后，相关区域内所有P1-P5任务暂停，机器人从最近点驰援。
- P1任务触发后，对应机器人完成当前动作后立即暂停当前任务，接受新P1任务。
- 单台机器人同时持有P0任务队列上限：1个；P1队列上限：2个；P2+合计队列上限：12个。

## 4. 路径规划算法

### 4.1 全局路径规划（A*改进版）

调度引擎全局路径规划采用**加权A\*算法**（Weighted A*），并结合**交通流预测模型**（基于历史人流密度数据）：

**代价函数：**

```
f(n) = g(n) + h_weighted(n)
h_weighted(n) = ε × h(n)    // ε = 1.2（加速收敛，允许5%次优）
总代价 = 路径物理距离代价 + 人流密度惩罚代价 + 设备冲突惩罚代价 + 禁区代价（∞）
```

**人流密度惩罚参数：**

| 区域人流密度 | 代价倍数 |
|---|---|
| < 0.2人/m² | ×1.0（无惩罚）|
| 0.2 ~ 0.5人/m² | ×1.3 |
| 0.5 ~ 1.0人/m² | ×2.0 |
| > 1.0人/m² | ×5.0（强制绕行）|
| 封控区域 | ×∞（绝对禁止）|

### 4.2 局部避障（TEB算法）

局部避障在**实时控制周期50ms**内完成：

| 参数 | 默认值 | 可调范围 |
|---|---|---|
| 规划时域（预测步数）| 15步 | 8~30步 |
| 时间分辨率 | 0.3s | 0.1~0.5s |
| 最小障碍物安全距离 | 0.35m | 0.2~0.8m |
| 人员最小安全距离 | 0.8m | 0.5~2.0m |
| 最大线速度 | 1.8m/s | 0.2~2.0m/s |
| 最大角速度 | 1.2rad/s | 0.5~2.0rad/s |
| 最大线加速度 | 0.5m/s² | 0.2~1.0m/s² |

## 5. 总控操作台操作规程

### 5.1 系统登录

1. 打开调度控制台（内网访问地址：`https://emis-dispatch.syzbq.internal:8443`）
2. 使用**EMIS专用账号**登录，账号格式：`dispatch_员工工号`（如：`dispatch_D00234`）
3. 进行**双因素认证**：输入账号密码后，通过员工手机的**EMIS Authenticator**App进行确认
4. 登录成功后，系统显示全园区机器人实时状态大屏
5. 确认当前轮班信息正确（轮班时间：A班06:00-14:00，B班14:00-22:00，C班22:00-06:00）

### 5.2 班次接管流程（SOP-DISPATCH-201）

| 步骤 | 操作 | 时间要求 |
|---|---|---|
| D201-01 | 登录系统，查看上一班次遗留任务队列 | 交班前15min |
| D201-02 | 确认所有在线机器人状态正常（在线率基准≥98.5%）| 交班前10min |
| D201-03 | 接收上班次调度员口头/文字交接，确认特殊任务/禁区 | 交班前5min |
| D201-04 | 双方在EMIS系统内完成电子签名交接确认 | 准时交班 |
| D201-05 | 接班调度员正式接管，上班次调度员降权为只读模式 | 交班后立即 |

### 5.3 手动派单操作

当自动调度无法满足需求时，调度员可手动创建任务：

1. 主界面点击【任务管理】>【新建手动任务】
2. 填写任务表单：
   - **任务类型**：从下拉菜单选择（安全巡检/货物核查/环境检测/设备检查/其他）
   - **目标区域**：在地图上点选或输入区域编码（格式：`BLDG-仓库号-楼层-区块`，如`BLDG-A03-01-SOUTH`）
   - **优先级**：P0~P5选择（P0/P1需填写原因，自动抄送上级）
   - **指派方式**：系统推荐/人工指定机器人编号
   - **到达要求**：最晚到达时间（精确到分钟）
3. 点击【提交】，系统在**3秒内**反馈任务接受确认及预计到达时间（ETA）
4. 若系统超过30秒未接受任务，触发错误码`DSP-E008`，需人工干预

### 5.4 紧急禁区操作

**当接到安全事件通知时（如火警、泄漏）：**

1. 在主界面地图上画定禁区范围（多边形选取）
2. 点击【设定禁区】>【紧急禁区】，禁区立即生效
3. 系统自动规划位于禁区内的机器人的撤离路径（最快路径优先）
4. 机器人撤离到达安全区后，系统发送通知
5. 禁区解除：需调度员+安全大队负责人双人确认后方可解除

## 6. 冲突与死锁解决机制

### 6.1 死锁检测

- 系统每**200ms**执行一次全局死锁检测（Chandy-Misra-Haas分布式快照算法）
- 死锁判定条件：**≥2台机器人互相等待对方让行，持续时间≥8秒**

### 6.2 死锁解决策略

系统按以下优先级顺序自动解决：

1. **优先级让行**：优先级低的机器人主动后退2m让行
2. **随机退让**：优先级相同时，序列号较大的机器人退让
3. **绕路重规划**：退让机器人选择备用路径绕行
4. **人工干预**：若自动解决失败（超过60秒），触发告警`DSP-E015`，调度员接管手动控制

## 8. 调度效率优化建议

### 8.1 日常操作建议

- **避免高峰期大批量派单**：14:00-16:00为货物作业高峰，尽量将P3以下任务延迟至17:00后
- **利用夜间窗口**：22:00-06:00为低人流窗口，应安排P5地图更新和设备深层检查任务
- **电量预留策略**：常规情况下，机器人电量低于30%时,调度员应主动发起充电任务，不应等待系统自动触发（8%阈值）

### 8.2 季节性调整

三亚高温季节（4月-10月），室外M3机器人补充参数：
- 续航时间缩短约15%（高温散热消耗），按实际续航调整充电频次
- 地面温度超过55°C时，限制连续作业时间≤90min/次，强制返站降温30min
