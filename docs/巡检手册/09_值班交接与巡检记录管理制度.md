# 值班交接与巡检记录管理制度

**文件编号：** SYZBQ-OPM-009  
**版本号：** V2.1.0  
**发布日期：** 2025年05月01日  
**编制部门：** 智能运营中心 · 运营管理组  
**审批人：** 智能运营中心主任 张晓华  
**密级：** 内部资料

---

## 1. 制度目的

规范EMIS系统各工作岗位的值班交接流程及巡检记录管理，确保：
1. 巡检任务的连续性和完整性，杜绝交接漏洞
2. 异常事件的信息完整传递，防止信息断链
3. 巡检数据记录的规范性、完整性和可追溯性
4. 满足海关审计和监管机构对运营记录的查阅要求

| 班次 | 时间段 | 交接时间缓冲期 |

### 2.3 节假日值班安排

- 法定节假日保持三班制运转，不得减员
- 节假日加班费按《三亚综合保税区薪酬管理办法》第18条执行（法定节假日×3，休息日×2）
- 春节（7天）、国庆（7天）期间，由部门主管提前14天排班并报人力资源部备案

## 3. 交接班流程（SOP-HANDOVER-901）

### 3.1 标准交接步骤

**接班人员须提前15分钟到岗，完整参与交接流程。**

| 步骤编号 | 步骤名称 | 操作内容 | 责任人 | 时间要求 |
|---|---|---|---|---|
| HO-01 | 系统状态核查 | 接班人查看EMIS大屏，确认所有在线机器人状态和当前任务队列数量 | 接班人 | 交班前15min |
| HO-02 | 异常事件交接 | 当班人逐一口述/文字描述当班期间发生的所有P0/P1级异常事件及处置状态 | 当班人 | 当班结束前 |
| HO-03 | 未完成任务交接 | 列出所有未完成或暂停的任务，说明原因和预期恢复时间 | 当班人 | 当班结束前 |
| HO-04 | 设备异常交接 | 告知接班人当前故障机器人编号、故障描述和维修状态（维修单编号） | 当班人 | 当班结束前 |
| HO-05 | 特殊指令交接 | 告知接班人当前生效的临时禁区、特殊任务或上级特别指令 | 当班人 | 当班结束前 |
| HO-06 | 电子签名交接 | 双方在EMIS系统中完成电子签名，系统自动记录交接时间 | 双方 | 准时 |
| HO-07 | 交接完成确认 | 接班人确认已掌握所有重要信息，正式接管权限 | 接班人 | 交接后立即 |

### 3.2 以下情况须降级/升级交接

**需延迟交接（当班人不得离岗）：**
- 当前存在**未处置的P0级事件**
- 机器人在线率低于**95%**（需运维组先恢复至正常水平）
- 调度系统出现`DSP-E001`（地图服务连接失败）等CRITICAL级错误

**可加速交接（30分钟内完成）：**
- 无异常事件，系统运行正常
- 任务完成率≥99%，无未完成P1+任务

## 4. 巡检记录管理

### 4.1 EMIS自动生成记录

系统自动记录以下数据，无需人工重复录入：

| 记录类型 | 自动生成触发条件 | 记录内容要素 | 格式 |
|---|---|---|---|
| 任务执行记录 | 每个巡检任务完成时 | 任务ID、机器人ID、区域、开始/结束时间、执行路径、采集数据摘要 | JSON |
| 货物识别记录 | 每次识别行为 | 识别时间、货物位置、识别结果、置信度、图片链接 | JSON |
| 告警事件记录 | 每次告警触发 | 时间、类型、位置、机器人、告警等级、处置结果 | JSON |
| 传感器采样记录 | 每30秒采集一次 | 温度、湿度、气体浓度等全量传感器数据 | InfluxDB时序数据 |
| 系统操作审计 | 每次人机交互 | 操作员、时间、操作类型、操作对象、操作结果 | JSON（不可篡改）|

### 4.2 人工记录要求

以下情况需要人工补充记录：

| 场景 | 记录工具 | 所需字段 |
|---|---|---|
| P0/P1事件处置过程 | EMIS事件记录模块 | 处置负责人、到场时间、处置措施、关闭时间、根因 |
| 机器人现场维修记录 | 维修单系统（MAINT-SYSTEM）| 维修人、故障描述、更换零件、维修时间、测试结果 |
| 货物移交记录 | FORM-ANO-001（纸质+扫描上传）| 移交双方签字、货物清单、移交时间、接收部门 |
| 校准记录 | EMIS-CalibTool自动生成 | 校准人、时间、结果、不合格项处置 |
| 人工复核结论 | 复核工作台 | 复核员工号、结论（通过/扣留/升级）、备注 |

### 4.3 记录保存期限

| 记录类别 | 电子记录保存期 | 纸质记录保存期 | 销毁审批要求 |
|---|---|---|---|
| 班次交接日志 | 5年 | 1年（原件归档）| IT+合规双签 |
| 任务执行记录（完成，无异常）| 3年 | 不要求 | — |
| 告警事件记录 | 10年 | — | IT+合规+法务三方签 |
| 货物识别记录（含图像）| 3年（图像90天热存储）| — | — |
| 人工复核记录 | 10年 | 3年（重要案件）| 法务合规签批 |
| 维修记录 | 设备服役期+3年 | — | — |

### 4.4 记录查阅权限

| 记录类型 | 内部可查阅岗位 | 外部可查阅机构 |
|---|---|---|
| 班次交接日志 | 调度员、中心主任、CISO | 安全审计机构（依法）|
| 货物识别记录 | 合规审计员、海关驻场、CISO | 海关（依法）、法院（依法）|
| 告警事件记录 | 合规审计员、安全组、中心主任 | 海关、警察、法院（依法）|
| 系统操作审计 | CISO、IT主管 | 监管机构（依法）|

## 5. 巡检记录质量要求与考核

| 考核指标 | 目标值 | 低于目标的处理 |
|---|---|---|
| 班次交接日志完整率 | 100% | 缺失一次，提醒；连续3次，计入绩效 |
| 人工记录及时率（按时限内完成）| ≥98% | 月度通报 |
| 记录准确率（无错误字段）| ≥99.5% | 错误超标，部门负责人出具改进说明 |
| 审计日志完整性（无缺失）| 100% | 触发S3安全事件处置 |

## 6. 系统错误码

| 错误码 | 描述 | 处置 |
|---|---|---|
| LOG-E001 | 交接日志提交失败（网络错误）| 手动填写纸质版，待系统恢复后补录 |
| LOG-E002 | 电子签名服务不可用 | 截图本地保存，联系IT恢复 |
| LOG-E003 | 任务记录写入数据库失败 | 本地缓存，IT告警 |
| LOG-E004 | 记录查询超时（超过3s）| 切换备用数据库节点查询 |
| LOG-E005 | 记录保存期满，未经审批删除防护触发 | 提醒数据管理员办理审批手续 |
