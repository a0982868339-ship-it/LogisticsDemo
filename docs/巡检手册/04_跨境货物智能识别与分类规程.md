## 1. 目的

规范三亚综合保税区具身智能巡检机器人对跨境货物进行自动识别、分类及合规判定的算法配置、操作流程及人工复核要求，确保货物识别准确率、合规判定的法律有效性及数据链的完整可追溯性。

## 2. 货物识别系统架构

### 2.1 识别流水线（识别管道）

```
原始图像采集 (60fps)
    │
    ▼
图像预处理（去噪/白平衡/畸变校正）
    │
    ▼
目标检测模型（YOLO-v9-Custom，实时检测货物/托盘/包装单元）
    │
    ▼
多模态融合（RGB + 深度 + 热成像）
    │
    ▼
货物特征提取（外包装特征/尺寸/重量估算/颜色/品牌LOGO）
    │
    ▼
OCR识别（运单号/条形码/QR码/中英文标签文字）
    │
    ▼
品类分类模型（ResNet-152+Transformer，覆盖8,420个HS编码类目）
    │
    ▼
合规规则引擎（实时匹配禁限商品清单/限额规则/监管要求）
    │
    ▼
置信度评估与分流（自动通过 / 人工复核 / 直接告警）
    │
    ▼
数据上链（区块链存证，哈希写入：SHA-256）
```

## 4. 识别合规判定规则

### 4.1 自动通过条件（PASS）

同时满足以下全部条件时，系统自动标记为"合规通过"：

1. 目标检测置信度 ≥ **0.92**
2. HS品类分类置信度 ≥ **0.88**
3. OCR运单号识别成功率 = **100%**（单次识别）
4. 识别货物品类不属于SEN-001至SEN-007任一子类
5. 识别品类与申报品类匹配度 ≥ **90%**（字符串相似度+语义相似度综合评分）
6. 货物外包装完整性评分 ≥ **0.80**（无明显破损）

### 4.2 人工复核条件（REVIEW）

满足以下任一条件时，进入人工复核队列：

| 触发条件 | 优先级 | 复核时限 |
|---|---|---|
| 目标检测置信度 0.75~0.92 | P3 | 30分钟内 |
| HS品类分类置信度 0.70~0.88 | P3 | 30分钟内 |
| OCR运单号识别失败（自动重试3次后）| P2 | 15分钟内 |
| 识别与申报品类匹配度 75%~90% | P2 | 15分钟内 |
| 货物外包装评分 0.50~0.80（疑似破损）| P3 | 30分钟内 |
| 货物属于SEN-001/SEN-002/SEN-003子类 | P1 | 5分钟内 |

### 4.3 告警拦截条件（ALERT）

满足以下任一条件时，立即触发P0/P1告警并停货：

| 触发条件 | 告警等级 | 联动动作 |
|---|---|---|
| 目标检测置信度 < 0.75，连续3次 | P1 | 机器人驻留，联系人工到场 |
| 识别与申报品类匹配度 < 75% | P1 | 停货标记，通知海关驻场 |
| 识别属于SEN-004~SEN-007任一子类 | P0 | 立即封堵区域+多部门联动通报 |
| 伪造标签检测置信度 > 0.85 | P1 | 停货+通知知识产权执法 |
| 货物有重量与申报差异 > ±20% | P1 | 停货，申请人工称重复核 |

## 5. 操作流程

### 5.1 例行巡检货物识别流程（SOP-ID-401）

```
[开始] → 机器人进入仓储区 → 启动货物巡检模式
    │
    ▼
扫描货架（点云+RGB融合）→ 生成货物分布地图（每5m一次全景拍摄）
    │
    ▼
系统自动对比申报货物清单（对接WMS系统，实时同步）
    │
    ▼
逐一识别采样货物（抽检率：标准区15%，高风险区30%，重点品类100%）
    │
    ▼
判定结果分流（PASS / REVIEW / ALERT）
    │           │                │
    ▼           ▼                ▼
  归档       加入复核队列     触发告警+人工到场
    │           │
    ▼           ▼
  继续下一货架   复核员处理（见5.2）
```

### 5.2 人工复核流程（SOP-ID-402）

1. 复核员收到系统推送的复核任务通知（手机App+PC工作台）
2. 打开复核工作台（`https://emis-review.syzbq.internal:8080`），查看机器人拍摄的全角度图像、视频及识别结论
3. 查阅对应申报单据（系统自动关联WCO-EDI申报数据）
4. 作出判定：
   - **确认合规**：点击【通过】，输入工号确认
   - **确认违规**：点击【扣留】，填写违规原因分类代码（见附录A），自动触发海关通报流程
   - **无法判定**：点击【升级】，上报海关驻场监管员现场核查
5. 所有操作留存审计日志，不可删除，保存期限：**10年**

## 7. 相关错误码

| 错误码 | 描述 | 严重等级 | 处置措施 |
|---|---|---|---|
| ID-E001 | WMS数据同步失败（拉取报关单超时）| 中等 | 重试3次后降级为仅视觉识别，标记货物待复核 |
| ID-E002 | HS分类模型推理超时（>2s）| 中等 | 切换轻量备用模型，异步重推理 |
| ID-E003 | 伪造标签检测模型离线 | 中等 | 所有SEN-001类货物自动转人工复核 |
| ID-E004 | 区块链存证写入失败 | 严重 | 本地缓存，等待链路恢复后补写；超2h未恢复则报告IT |
| ID-E005 | 人工复核超时（超过规定时限未处理）| 低 | 系统升级任务优先级，自动抄送部门主管 |
| ID-E006 | 申报单据查询无结果 | 中等 | 标记为"无申报记录货物"，触发P1告警 |
| ID-E007 | GPU推理内存溢出 | 严重 | 重启推理服务，通知该区域暂停AI识别、转人工 |
| ID-E008 | 识别图像质量低于可用阈值（模糊/遮挡）| 低 | 机器人重新拍摄（最多3次），失败则人工复核 |
| ID-E009 | 海关接口服务不可用 | 严重 | 本地队列暂存，待接口恢复后批量上报；同时通知海关驻场 |
