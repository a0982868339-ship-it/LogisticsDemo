# 数据安全与隐私保护管理规程

**文件编号：** SYZBQ-SEC-008  
**版本号：** V2.3.0  
**发布日期：** 2025年06月01日  
**编制部门：** 信息技术部 · 网络安全组 / 法务合规部  
**审批人：** 首席信息安全官（CISO）孙伟明  
**密级：** 内部资料 · 机密

## 1. 目的与适用范围

### 1.1 目的

本规程旨在保护三亚综合保税区EMIS系统在运营过程中产生、收集、传输、存储和使用的所有数据的安全性，防止数据泄露、篡改、丢失或未经授权访问，满足国家法律法规和行业合规要求。

### 1.2 适用范围

适用于：
- 所有使用EMIS系统或具有数据系统访问权限的员工（约9,760人）
- 所有外包服务商及合作伙伴（须签署《数据保护补充协议》）
- 机器人采集的所有数据（图像、视频、传感器数据、行为记录等）

---

## 2. 数据分级与保护要求

### 2.1 数据分级框架

本园区采用**四级数据分级体系**，遵循GB/T 39477-2020《信息安全技术政务信息共享数据安全技术要求》：

| 级别 | 名称 | 定义 | 典型数据示例 | 泄露后果评估 |
|---|---|---|---|---|
| **L4** | 绝密数据 | 泄露可能危害国家安全或重大公共利益 | 海关执法策略、内控算法源码、国家安全部门调取数据 | 极严重 |
| **L3** | 机密数据 | 泄露可能造成重大商业损失或违反法律 | 进口商信息、未申报货物证据、完整人员行为轨迹、商业合同 | 严重 |
| **L2** | 受限数据 | 泄露可能影响业务运营或个人隐私 | 巡检视频片段、人员通行记录、告警事件记录、设备状态数据 | 中等 |
| **L1** | 公开数据 | 可以公开或不涉及实质敏感内容 | 园区平面图（公开版）、机器人编号列表、日常KPI汇报数据 | 轻微 |

### 2.2 各级别保护要求

| 保护要求 | L4 | L3 | L2 | L1 |
|---|---|---|---|---|
| 传输加密 | TLS 1.3 + 国密SM4 | TLS 1.3 | TLS 1.3 | HTTPS |
| 存储加密 | AES-256-GCM + 国密SM4双加密 | AES-256-GCM | AES-128 | 可选 |
| 访问控制 | 最小权限 + 双人审批 | 最小权限 + 审批 | 基于角色（RBAC）| 登录即可 |
| 日志审计 | 全量记录，实时审查 | 全量记录 | 抽样审计 | 保留90天 |
| 数据脱敏 | 显示时强制脱敏 | 查看需审批，展示脱敏 | 部分脱敏 | 无要求 |
| 存储期限 | 永久（含离线备份）| 10年 | 3年 | 1年 |
| 跨境传输 | 禁止（须专项审批）| 须安全评估+合规备案 | 须备案 | 无限制 |

## 3. 个人信息保护要求

### 3.1 机器人采集的个人信息类型

| 信息类型 | 采集方式 | 是否属于个人信息 | 处理要求 |
|---|---|---|---|
| 人脸图像 | RGB相机 | **是（生物识别）**| 通知同意，最小化采集，不用于人脸识别（仅行为分析） |
| 人体姿态 | 深度相机+点云 | **是（敏感个人信息）**| 匿名化处理，去除身份关联 |
| 员工工号（徽章）| OCR识别 | **是** | 仅记录通行合规，不建立行为画像 |
| 车辆车牌 | 车牌识别相机 | **是** | 进出时记录，30天后自动删除 |
| 语音交互 | 麦克风阵列 | **是（需特别保护）**| 仅本地处理，不上云，不存储原始音频超过72小时 |

### 3.2 个人信息处理原则（遵循《个人信息保护法》第六条）

1. **最小必要原则**：机器人仅采集完成巡检任务所必需的最少量个人信息
2. **目的限制原则**：采集的个人信息仅用于巡检合规目的，不用于商业分析或其他目的
3. **知情同意原则**：员工入职时通过《EMIS系统个人信息处理告知书》（FORM-PRIVACY-001）知情同意，外来访客在园区入口处以弹窗+告示牌形式告知
4. **存储限制原则**：达到保存期限后自动删除（系统自动执行数据生命周期管理）
5. **安全保护原则**：按数据分级要求加密存储和传输

## 4. 访问控制管理

### 4.1 账号管理规定

| 规定项目 | 要求 |
|---|---|
| 账号申请 | 填写FORM-IAM-001，部门主管+IT审批，双人确认 |
| 账号命名规则 | `部门代码_员工工号`（如：`DISP_D00234`）|
| 密码策略 | 长度≥12位，含大小写字母+数字+特殊字符,90天强制更换 |
| 多因素认证（MFA）| 所有L2以上系统**强制MFA**，使用EMIS Authenticator App |
| 账号锁定策略 | 连续5次错误登录锁定30分钟；IT解锁需人工审批 |
| 特权账号管理 | 超级管理员账号由CISO控制，每次使用须审批，双人操作 |
| 账号注销 | 员工离职当日16:00前注销，外包人员项目结束后48h内注销 |
| 账号审计 | 每季度开展一次账号存活性审计，发现异常账号立即锁定 |

### 4.2 权限模型（RBAC）

| 角色 | 数据访问级别 | 系统功能权限 | 人数（约）|
|---|---|---|---|
| 超级管理员 | L1~L4全量 | 全部 | 3人（CISO+2名副手）|
| 系统管理员 | L1~L3 | 系统配置、账号管理 | 8人 |
| 调度员 | L1~L2 | 调度操作、任务管理、告警处理 | 32人 |
| 合规审计员 | L1~L3 | 复核操作、报告查看、海关接口操作 | 24人 |
| AI算法工程师 | L1~L2（匿名数据）| 模型训练、算法配置 | 12人 |
| 运维工程师 | L1~L2 | 设备维护、校准工具 | 24人 |
| 数据分析师 | L1~L2（脱敏）| 报表查看、数据导出（受限）| 6人 |
| 海关驻场人员 | L2~L3（指定字段）| 复核查看、接收通报 | 120人 |
| 普通员工（只读）| L1 | 仅查看公开报表 | ~9,500人 |

## 5. 数据传输安全

### 5.1 加密配置要求

```yaml
# EMIS系统TLS配置标准（nginx/HAProxy）
ssl_protocols:              TLSv1.3;  # 禁止低版本
ssl_ciphers:                TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256;
ssl_prefer_server_ciphers:  on;
ssl_session_timeout:        1d;
ssl_session_cache:          shared:SSL:50m;
ssl_certificate:            /etc/ssl/emis-cert.pem;
ssl_certificate_key:        /etc/ssl/emis-key.pem;  # 存储于HSM，不落盘
ssl_client_certificate:     /etc/ssl/ca-bundle.pem; # mTLS CA证书
ssl_verify_client:          on;  # 对所有外部接口
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

### 5.2 数据脱敏规则

| 字段类型 | 脱敏方式 | 示例 |
|---|---|---|
| 姓名 | 保留第一字，其余替换为* | 张** |
| 手机号 | 保留前3位和后4位 | 138****5678 |
| 身份证号 | 保留前4位和后4位 | 4401****1234 |
| 银行卡号 | 仅保留后4位 | ****9088 |
| 邮箱 | 用户名部分脱敏 | zh***@example.com |
| IP地址 | 最后一组遮掩 | 192.168.1.*** |
| 车牌号 | 保留省份和后2位 | 粤B***89 |

## 6. 安全事件响应

### 6.1 安全事件等级

| 等级 | 定义 | 响应时限 | 通报要求 |
|---|---|---|---|
| **S1（特大）** | 数据大规模泄露，影响≥10万条个人信息；系统完全不可用 | 立即响应 | 1小时内上报CISO；24小时内上报主管部门（工信部/网信办）|
| **S2（重大）** | L3级以上数据泄露；核心业务停摆>2小时 | 1小时内 | 4小时内上报CISO；48小时内上报主管部门 |
| **S3（严重）** | L2数据异常访问；单一业务模块停摆>4小时 | 4小时内 | 24小时内上报IT主管及法务合规部 |
| **S4（一般）** | 违反数据使用规范但未实际泄露；服务轻微降级 | 24小时内 | 周报汇总 |

### 6.2 响应流程

```
发现安全事件
    │
    ▼
立即通知CISO（7×24小时热线：98-9100）
    │
    ▼
事件初步评估（≤30分钟）：确认影响范围、数据类型、原因
    │
    ▼
S1/S2级 → 启动安全应急预案（ERP-SEC-001）
S3/S4级 → 按部门SOP处置
    │
    ▼
遏制措施：隔离受影响系统/账号/数据，防止事件扩大
    │
    ▼
取证：系统日志、操作记录、网络流量截取（不得覆盖或删除）
    │
    ▼
恢复：在确认安全后逐步恢复系统
    │
    ▼
溯源分析：根因分析（RCA），形成事件报告（72小时内完成）
    │
    ▼
改进措施：评估并更新安全策略、技术控制及人员培训
```

## 8. 安全违规处罚规定

| 违规行为 | 处罚级别 | 具体处罚 |
|---|---|---|
| 未授权访问L3/L4数据 | 严重 | 立即停职调查；情节严重解除劳动合同，移送司法机关 |
| 将系统数据复制至个人设备 | 严重 | 警告+停职30天；造成泄露则解除合同 |
| 账号共享/密码外借 | 中等 | 书面警告，计入年度绩效；再次违规加重处罚 |
| 故意规避MFA认证 | 严重 | 立即停用账号，纪律处分 |
| 未按规定销毁含敏感数据的纸质文件 | 轻微 | 口头警告，强制参加数据安全培训 |
| 向外部人员泄露系统口令 | 严重 | 解除劳动合同，依法追责 |

## 9. 安全错误码

| 错误码 | 描述 | 处置 |
|---|---|---|
| SEC-E001 | JWT令牌格式无效 | 引导重新登录 |
| SEC-E002 | 令牌已过期（>2小时）| 刷新令牌 |
| SEC-E003 | MFA验证失败 | 限制3次，超出锁定账号 |
| SEC-E004 | IP地址不在允许范围 | 拒绝访问，记录告警 |
| SEC-E005 | 数据访问超出权限范围 | 拒绝访问，记录审计，通知安全团队 |
| SEC-E006 | 加密模块不可用 | 拒绝数据写入，通知IT |
| SEC-E007 | 审计日志写入失败 | 系统降级，优先保证日志完整性 |
| SEC-E008 | 证书链验证失败 | 拒绝连接，检查CA配置 |
| SEC-E009 | 数据脱敏服务不可用 | 暂停L3+数据查询，通知IT |
| SEC-E010 | 异常登录地检测（首次境外IP）| 触发二次验证，通知CISO |
