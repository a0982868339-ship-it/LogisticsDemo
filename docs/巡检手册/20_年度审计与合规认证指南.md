## 2. 海关监管合规年度审计

### 2.1 审计时间与准备

| 时间节点 | 工作内容 | 责任人 |
|---|---|---|
| 每年10月1日前 | 与三亚海关协商确认当年审计时间窗口 | 合规审计部部长 |
| 审计前8周 | 启动内部自查（见2.2节）| 合规审计部 |
| 审计前4周 | 完成自查报告，提交整改（如有问题）| 合规审计部 |
| 审计前2周 | 准备审计材料清单，预约EMIS系统演示 | 合规审计部+IT |
| 审计当日 | 协助海关审计员开展审计，提供系统访问权限 | 合规审计部+IT+总经理室 |
| 审计后2周 | 接收审计意见书，制定整改计划 | 合规审计部+法务 |
| 审计后4周 | 完成整改并向三亚海关提交整改报告 | 合规审计部 |

### 2.2 内部自查清单（海关审计方向）

**A. 巡检数据完整性核查**

| 检查项目 | 方法 | 合格标准 |
|---|---|---|
| 全年任务执行记录是否完整 | 查询EMIS数据库，核对任务总量与记录数 | 记录完整率=100% |
| 异常告警记录是否全部有处置结论 | 检索所有P0/P1告警，确认有处置结论字段 | 闭环率≥99.9% |
| 海关通报是否及时（≤30分钟）| 统计CUS-API-002调用时间与告警触发时间之差 | 及时率≥99% |
| 货物抽检率是否满足合同要求 | 按区域统计全年平均抽检率 | 各区域均满足最低要求 |
| 区块链存证是否连续（无断链）| 检查HNTradeChain交易记录连续性 | 存证率=100% |

**B. 数据真实性核查**

- 随机抽取100条巡检记录，核实EMIS数据与机器人原始视频/图像的一致性
- 核查告警记录中的置信度是否存在批量异常修改迹象（数据库操作日志审查）
- 核查是否存在"只有告警记录但无机器人作业记录"的数据孤立情况

**C. 人员合规核查**

- 检查所有合规审计员是否持有效认证
- 检查所有海关通报是否有经授权人员签名
- 核查权限日志中是否存在未授权的海关接口操作

## 3. 信息安全等级保护（等保2.0）年度测评

### 3.1 等保三级测评要求

EMIS系统定级为**等保三级（S3A3G3）**，须每年开展一次测评。

**测评机构要求**：具有公安部颁发的等保测评资质（甲级）

**年度测评周期**（典型时间表）：

| 时间 | 阶段 | 活动内容 |
|---|---|---|
| T-12周 | 准备阶段 | 更新系统定级备案；组织内部差距分析 |
| T-10周 | 整改阶段 | 发现并整改任何与等保要求的差距 |
| T-8周 | 测评联络 | 与测评机构签合同，协商测评时间和范围 |
| T-2周 | 材料准备 | 准备技术文档、网络拓扑、设备清单、制度文件 |
| T~T+5天 | 现场测评 | 测评机构进场，检查文档+工具测试+访谈 |
| T+3周 | 整改期 | 接收测评报告草稿，整改各类问题 |
| T+6周 | 出具报告 | 测评机构出具正式报告（结论：优/良/中/差）|
| T+8周 | 备案 | 向属地公安机关（海南省公安厅）提交备案材料 |

**当前等保情况**：EMIS系统等保三级测评结论（2024年）：**良（87分/100分）**

### 3.2 等保关键技术指标自检

| 等保项目 | 检查项 | 当前状态 |
|---|---|---|
| 安全通信 | 传输加密（TLS 1.3）| ✅ 已实施 |
| 访问控制 | RBAC+MFA | ✅ 已实施（全系统强制MFA）|
| 审计日志 | 不可删除操作日志，保存6个月以上 | ✅ 保存10年 |
| 入侵防范 | WAF+IDS/IPS | ✅ 已部署（Cloudflare WAF + Suricata）|
| 恶意代码防护 | 终端防病毒、容器扫描 | ✅ 已实施 |
| 数据备份 | 每日全量备份，异地容灾 | ✅ 异地备份（海口数据中心）|
| 密码管理 | 密码复杂度+定期更换 | ✅ 策略已实施 |
| 个人信息保护 | 脱敏处理、最小采集 | ✅ 已实施 |

## 4. 数据保护年度合规评估

### 4.1 个人信息保护影响评估（DPIA）

根据《个人信息保护法》第55条，以下EMIS处理活动须进行DPIA：

| 处理活动 | DPIA Required | 上次评估日期 | 下次评估不晚于 |
|---|---|---|---|
| 机器人人脸/人体图像采集 | 是 | 2024-11-01 | 2025-11-01 |
| 员工通行记录处理 | 是 | 2024-11-01 | 2025-11-01 |
| 车辆/车牌识别 | 是 | 2024-11-01 | 2025-11-01 |
| 海关申报数据处理 | 是 | 2024-11-01 | 2025-11-01 |
| HNFTZ-DC数据共享 | 是（数据共享评估）| 2024-11-01 | 2025-11-01 |

**DPIA评估内容（每次评估）：**
1. 处理目的与法律依据是否仍然有效
2. 最小必要性原则是否满足（数据字段是否有新增且必要性未评估的）
3. 技术安全措施是否仍然有效（加密、访问控制等）
4. 数据留存期限是否符合规定
5. 个人权利行使机制是否完善（访问权、删除权、可携带权）
6. 是否发生数据泄露及其影响

### 4.2 数据主体权利响应记录

| 权利类型 | 响应时限（法律要求）| 处理流程 | 2024年请求数 |
|---|---|---|---|
| 访问权（查看个人数据）| 15个工作日 | 员工通过OA提交→CISO审批→IT导出脱敏数据 | 24次 |
| 更正权（纠正错误数据）| 15个工作日 | 同上，附修正数据 | 8次 |
| 删除权（擦除数据）| 15个工作日 | 须满足删除条件才可删 | 3次（均驳回，在保存期内）|
| 限制处理权 | 立即生效 | 系统标记"限制处理"标签 | 0次 |
| 数据可携带权 | 30个工作日 | 导出标准格式数据（JSON/CSV）| 2次 |

## 7. 审计问题跟踪

所有审计发现的问题须在EMIS问题跟踪系统（`issues.syzbq.internal`，使用Jira）中创建问题单，格式：

| 字段 | 说明 |
|---|---|
| 问题编号 | AUDIT-YYYY-NNN（如AUDIT-2025-042）|
| 审计来源 | 海关审计/等保测评/ISO/内部审计/DPIA |
| 问题描述 | 发现的具体问题（清晰描述、引用标准条款）|
| 严重/等级 | 严重缺陷/一般缺陷/观察项 |
| 整改措施 | 具体整改方案（可量化）|
| 责任人 | 部门主管 |
| 计划完成日期 | 法定或协商的整改期限 |
| 实际完成日期 | 由责任人在完成后填写 |
| 验证人 | 审计部门确认整改有效性 |
| 状态 | Open / In Progress / Resolved / Verified / Closed |
